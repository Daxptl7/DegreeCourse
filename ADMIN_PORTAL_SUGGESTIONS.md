# Admin Portal Suggestions for UniLearn PDEU

## Overview
The Admin Portal for UniLearn PDEU should serve as the central hub for verification, maintenance, and platform growth analytics.

## Suggested Sections

### 1. Dashboard & Reports
- **User Metrics**: Total registrations (Active/Inactive).
- **Course Engagement**: Enrollment rates, most popular categories.
- **System Health**: Error logs, server uptime.
- **Monthly Growth**: Visualizing user and course growth.

### 2. User Verification Queue
- **Teacher Approval**: Review profiles and credentials of new teacher registrations.
- **Student Management**: Filter students by School (SOT, SPT, etc.).
- **Staff Access Control**: Manage permissions for departmental admins.

### 3. Course Moderation
- **Approval System**: All courses should start in a `PENDING` state for admin review.
- **Resource Audit**: Verify that course materials comply with university standards.
- **Featured Content**: Promote important courses to the home page carousel.

### 4. Communication Tools
- **Platform-wide Alerts**: Send banners for system maintenance or deadlines.
- **Audit Logs**: Trace all administrative actions (Who changed what role?).
- **Feedback Loop**: Aggregated student reports on content.

### 5. Organizational Config
- **School List**: Manage the list of active PDEU schools.
- **Category Mapper**: Group courses under university-specific departments.
